Introduction: Pourquoi Claude Code?

La plupart des gens utilisent ChatGPT ou Claude en mode "chat" - ils doivent re-expliquer leur contexte a chaque conversation. Claude Code change completement la donne.

üí° La difference Claude Code

Claude Code tourne dans votre terminal et a acces a vos fichiers locaux. Il peut lire vos transcripts, vos notes, vos playbooks - et s'en souvenir pour vous aider de maniere contextuelle.

Cas d'usage GTM concrets

  • "Resume-moi le call avec Doctolib d'hier" - Claude lit le transcript et extrait les points cles
  • "Quelles objections sont revenues cette semaine?" - Analyse de tous vos transcripts recents
  • "Ecris un follow-up pour ce prospect" - Base sur le contexte du call + votre playbook
  • "Prepare-moi pour le call avec Alan demain" - Research + historique des interactions

Partie 3: Le fichier CLAUDE.md

Le fichier CLAUDE.md est le plus important. C'est la memoire permanente de Claude - il le lit automatiquement a chaque session.

‚ö†Ô∏è Placement critique

Le fichier CLAUDE.md doit etre a la racine du dossier ou vous lancez Claude Code. Il est lu automatiquement au demarrage.

3.1 Structure du CLAUDE.md

1

Creer le fichier CLAUDE.md

Voici un template complet pour GTM:

# CLAUDE.md - Contexte GTM

## Qui je suis
Je suis [Ton nom], GTM Engineer chez [Entreprise].
Mon role: generer des meetings qualifies pour l'equipe sales.

## Mon ICP (Ideal Customer Profile)
- **Secteur:** SaaS B2B, Fintech, HealthTech
- **Taille:** 50-500 employes
- **Geo:** France, Benelux
- **Personas cibles:** CTO, VP Engineering, Head of Growth
- **Signaux d'achat:** Levee de fonds recente, recrutement tech, nouveau CTO

## Stack actuel
- **Enrichissement:** Clay + Apollo
- **Outreach LinkedIn:** HeyReach
- **Cold email:** Smartlead
- **CRM:** HubSpot
- **Transcription:** Meetily

## Structure des fichiers
- `/transcripts/` - Transcripts de mes calls (format: YYYY-MM-DD_company_type.md)
- `/accounts/` - Fiches compte par entreprise
- `/playbooks/` - Scripts et templates
- `/competitive/` - Intel sur les concurrents
- `/weekly_notes/` - Notes hebdomadaires

## Comment m'aider

### Avant un call
- Lis la fiche compte dans /accounts/
- Lis les transcripts precedents avec cette entreprise
- Prepare 3-5 questions de discovery basees sur le contexte

### Apres un call
- Resume les points cles du transcript
- Identifie les objections et next steps
- Redige un email de follow-up

### Analyse hebdo
- Quelles objections reviennent le plus?
- Quels comptes progressent/stagnent?
- Suggestions d'actions

## Ton style de communication
- Direct et actionnable
- Pas de fluff ou de phrases generiques
- Propose des actions concretes
- Utilise le contexte des fichiers, pas des reponses generiques
IMAGE 09

Le CLAUDE.md definit le contexte permanent pour toutes vos sessions

2

Personnaliser selon votre contexte

Adaptez les sections a votre realite: votre ICP, vos outils, votre facon de travailler.

‚úÖ Best practice

Mettez a jour votre CLAUDE.md regulierement. Ajoutez les patterns que vous remarquez, les preferences que vous developpez.
